## 1. 软件概述

### 1.1 软件名称

基于深度学习的离散手语识别系统 V1.0  
英文名称：Deep Learning Based Discrete Sign Language Recognition System V1.0

### 1.2 建设目标

本系统用于对离散手语视频流进行自动识别，支持以下三类输入场景：

- 图片识别：对单张手语图像进行分类识别。  
- 视频识别：对上传视频按时间间隔切片识别，并输出“时间段-字符”结果。  
- 实时摄像头识别：对摄像头输入进行持续识别，输出累计文本与时间段结果。

### 1.3 典型适用场景

- 教学演示与课堂实训。  
- 手语识别算法原型验证。  
- 软件著作权登记展示材料。  
- 数据集标注辅助与结果快速浏览。

### 1.4 软件特点

- 基于 PyTorch 推理，支持 CPU/GPU。  
- 采用 MediaPipe 提取手部与上半身关键点。  
- 采用 Gradio Blocks 构建可视化界面。  
- 支持 Top-K 概率输出与时间片段结果。  
- 支持模型权重路径与词表路径动态配置。

## 3. 系统架构与模块说明

### 3.1 总体架构

系统采用“UI 层 + 推理服务层 + 特征提取层 + 模型层 + 结果展示层”的结构：

1. UI 层：Gradio Blocks 页面，负责交互。  
2. 推理服务层：`InferenceEngine`，负责加载模型与词表并缓存。  
3. 特征提取层：MediaPipe Hands + Pose 提取关键点。  
4. 模型层：`SignBiLSTMBaseline`，输出分类概率。  
5. 结果展示层：Top1/Top-K、时序文本、时间段表格。

### 3.2 模块划分

#### 模块 A：图片识别模块

- 输入：单张图片（RGB）。  
- 处理：关键点提取 -> 构造固定长度时序 -> 模型推理。  
- 输出：Top1、Top-K、置信度、推理耗时。

#### 模块 B：视频识别模块

- 输入：视频文件。  
- 处理：按时间间隔切段 -> 每段关键点提取 -> 模型推理 -> 段级结果聚合。  
- 输出：Top1、Top-K、推理耗时、时序文本、时间段结果表。

#### 模块 C：实时摄像头模块

- 输入：浏览器摄像头流。  
- 处理：会话状态维护（开始/停止/清空）-> 按时间间隔累计片段 -> 实时推理。  
- 输出：当前预测、Top-K、累计文本、时间段结果表。

### 3.3 关键组件说明

- `gradio_app.py`：系统入口，负责 UI 和推理流程串联。  
- `infer.py`：提供核心推理函数（复用其模型构建与推理逻辑）。  
- `src/slr_baseline/model.py`：模型定义。  
- `src/slr_baseline/keypoints.py`：关键点提取工具。  
- `src/slr_baseline/features.py`：特征构建逻辑。  
- `scripts/*.py`：数据扫描、元信息生成、关键点提取、可视化辅助脚本。

### 3.4 数据流说明

#### 图片识别数据流

图片输入 -> RGB 转换 -> 关键点提取 -> 构造固定 T 长度序列 -> 模型推理 -> UI 展示

#### 视频识别数据流

视频输入 -> 解码帧序列 -> 按 interval 切段 -> 每段提取关键点 -> 每段推理 -> 段级聚合 -> UI 展示

#### 摄像头识别数据流

摄像头帧 -> 会话状态缓存 -> 到达时间间隔阈值 -> 片段推理 -> 累计文本/时间表更新 -> UI 展示

## 4. 安装部署与运行步骤

### 4.1 获取代码

```bash
cd /home/Turtledove
cd Sign-Language-Trans
```

### 4.2 创建并激活环境（推荐 conda）

```bash
conda create -n Sign-Language python=3.10 -y
conda activate Sign-Language
```

### 4.3 安装依赖

```bash
pip install -r requirements.txt
```

### 4.4 启动系统

```bash
python gradio_app.py
```

默认监听：

- 地址：`127.0.0.1`  
- 端口：`7860`

可选参数：

```bash
python gradio_app.py --host 0.0.0.0 --port 7860 --share
```

参数说明：

- `--host`：服务绑定地址。  
- `--port`：服务端口。  
- `--share`：开启公网临时链接（Gradio）。  
- `--no-browser`：不自动打开浏览器。

### 4.5 首次运行检查项

- `checkpoints/best.pt` 是否存在。  
- `meta/vocab_gloss.json` 是否存在。  
- 浏览器是否允许摄像头权限（实时模式）。  
- 若使用 GPU，`torch.cuda.is_available()` 是否为 True。

### 4.6 数据准备与训练命令（目录规范化后）

```bash
# 1) 扫描数据并生成报告
python scripts/scan_csl.py --csl-root ./csl --report-path docs/report.md

# 2) 生成词表与清单
python scripts/prepare_meta.py --csl-root ./csl --meta-dir meta --seed 3407

# 3) 提取关键点
python scripts/extract_keypoints.py \
  --manifest meta/manifest.csv \
  --processed-root dataset_processed \
  --num-frames 32 \
  --num-workers 4

# 4) 可视化抽样
python scripts/visualize_samples.py \
  --manifest meta/manifest.csv \
  --processed-root dataset_processed \
  --output-dir outputs/debug_vis \
  --num-samples 20 \
  --seed 3407

# 5) 训练
python train.py \
  --manifest meta/manifest.csv \
  --processed-root dataset_processed \
  --vocab meta/vocab_gloss.json \
  --ckpt-dir checkpoints

# 6) 离线推理（默认权重 checkpoints/best.pt）
python infer.py \
  --input <视频文件或帧目录> \
  --checkpoint checkpoints/best.pt \
  --vocab meta/vocab_gloss.json \
  --topk 5
```

## 5. 功能与操作流程

### 5.1 顶部区域

页面顶部包含：

- 系统标题。  
- 子标题说明。  
- 左侧 Logo 标识。  
- 模型配置折叠面板（权重路径、词表路径、设备）。

### 5.2 图片识别 Tab 操作流程

步骤：

1. 进入“图片识别”页。  
2. 上传图片。  
3. 调整 `Top-K`。  
4. 点击“开始识别”。  
5. 查看右侧结果：
   - Top1 类别
   - 置信度
   - Top-K 列表
   - 推理耗时

### 5.3 视频识别 Tab 操作流程

步骤：

1. 进入“视频识别”页。  
2. 上传视频文件。  
3. 设置“时间间隔（秒）”。  
4. 设置 `Top-K`。  
5. 点击“开始识别”。  
6. 查看结果：
   - Top1 类别与置信度  
   - Top-K 列表  
   - 推理耗时  
   - 时序识别文本（按片段拼接）  
   - 视频时间片段结果表（序号、时间段、字符、置信度）

### 5.4 实时摄像头 Tab 操作流程

步骤：

1. 进入“实时摄像头”页。  
2. 允许浏览器访问摄像头。  
3. 设置“输出时间间隔（秒）”。  
4. 点击“开始会话”。  
5. 系统将持续累积识别片段并更新：
   - 当前 Top1/Top-K  
   - 累计文本  
   - 时间段结果表  
6. 可点击“停止会话”暂停累积。  
7. 可点击“清空会话”清理历史结果。  
8. 若自动更新不稳定，可点击“手动识别当前帧”。

### 5.5 结果解读建议

- Top1：当前时刻或当前片段最可能类别。  
- Top-K：给出候选分布，用于人工核查。  
- 时间段表：用于回看视频/实时流中每个时间窗的识别结果。  
- 累计文本：用于快速浏览整段输入的大致语义序列。

## 6. 输入输出说明

### 6.1 输入格式

- 图片：常见图像格式（JPG/PNG 等）。  
- 视频：常见视频格式（MP4/AVI/MOV 等，取决于 OpenCV 解码能力）。  
- 摄像头：浏览器实时帧流。

### 6.2 输出字段定义

#### Top-K 列表字段

- 排名：从 1 开始。  
- 类别：识别标签（gloss）。  
- 置信度：概率值（0~1）。

#### 时间段结果表字段

- 序号：片段编号。  
- 时间段：`start-end` 秒。  
- 字符：该片段预测类别。  
- 置信度：该片段 Top1 置信度。

### 6.3 模型配置输入

- Checkpoint 路径：`.pt/.pth` 权重文件路径。  
- Vocab 路径：词表 JSON 路径。  
- 推理设备：`auto` / `cpu` / `cuda`。

## 7. 异常处理与提示

### 7.1 常见异常与处理

| 现象 | 可能原因 | 处理建议 |
|---|---|---|
| 启动时报 `ModuleNotFoundError` | 依赖未安装 | 执行 `pip install -r requirements.txt` |
| 报 `Checkpoint not found` | 权重路径错误 | 检查并修正 checkpoint 路径 |
| 报 `Vocab not found` | 词表路径错误 | 检查并修正 vocab 路径 |
| 摄像头无画面 | 浏览器权限未开启 | 允许 Camera 权限，刷新页面 |
| 视频识别很慢 | 视频过长或无 GPU | 增大时间间隔，改用 CUDA |
| 报 MediaPipe API 错误 | mediapipe 版本不匹配 | 固定 `mediapipe==0.10.14` |
| 输出结果与预期偏差 | 模型泛化不足/数据域差异 | 使用同域数据微调并重训 |

### 7.2 运行时提示策略

系统在推理失败时会在结果区域显示：

- `推理失败: <异常信息>`  

会话模式提供状态提示：

- `会话状态：未开始`  
- `会话状态：运行中`  
- `会话状态：已停止`  
- `会话状态：已清空（未开始）`

### 7.3 排错顺序建议

1. 先排依赖问题。  
2. 再排路径问题（权重、词表、输入文件）。  
3. 再排设备问题（CPU/GPU、显存）。  
4. 最后排模型效果问题（数据域、训练质量）。

## 8. 运维建议与性能说明

### 8.1 性能建议

- 图片识别：单图可快速返回。  
- 视频识别：时间间隔越小，片段越多，耗时越大。  
- 实时识别：建议间隔设为 `0.5s~1.5s` 平衡稳定性与实时性。

### 8.2 资源建议

- CPU 推理：适合演示与小规模使用。  
- GPU 推理：适合实时性要求高的场景。  
- 建议在生产部署时限制并发，避免显存冲突。

### 8.3 日常维护

- 定期备份权重文件和词表。  
- 记录版本号与训练配置。  
- 新增类别后同步更新词表与模型。

## 9. 安全与合规说明

### 9.1 数据安全

- 输入视频与摄像头画面可能包含敏感信息，建议在内网使用。  
- 如用于生产，应增加访问控制与日志审计。

### 9.2 隐私合规

- 使用摄像头前应取得用户授权。  
- 禁止在未告知场景下采集与保存个人图像信息。

### 9.3 模型合规

- 模型输出仅为算法预测结果，不应作为唯一决策依据。  
- 涉及教育、政务、医疗等场景应配套人工复核流程。
